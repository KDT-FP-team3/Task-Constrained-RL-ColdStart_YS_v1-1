# Chainers Master Fund — Task-Constrained RL Cold-Start


로컬 실행 방법을 간단히 안내해 드립니다.

1. 환경 준비 (최초 1회)

cd c:\yssong\KDT-FT-team3-Chainers\Task-Constrained-RL-ColdStart_YS_v1-1
pip install -r requirements.txt
2. 앱 실행

streamlit run app.py
브라우저가 자동으로 열리거나 http://localhost:8501 접속

3. 시뮬레이션 절차
멤버 탭 선택 → 시뮬레이션할 종목 탭 클릭
파라미터 설정 (상단 6개 슬라이더)
Trading Days: 500, Train Episodes: 300
Auto Run Count: 10, Sim Min Steps: 30, Sim Step Mult: 10
Simulation 버튼 클릭 → PG Actor-Critic 파라미터 탐색 시작
완료 후 → "저장 및 반영" 버튼이 나타남
오른쪽에 저장 예정 파라미터 미리보기 표시
저장 및 반영 클릭 → 해당 멤버의 config.py 자동 업데이트
4. 저장 후 배포

git add members/member_N/config.py
git commit -m "optimize member N params"
git push
웹 앱은 업데이트된 config.py를 기본값으로 사용하므로, 시뮬레이션 없이도 최적 파라미터로 Run Evaluation 가능합니다.

참고: GPU가 있어도 현재 코드는 CPU 기반 Q-Learning/Actor-Critic이라 GPU 가속은 없습니다. 빠른 테스트가 필요하면 Train Episodes를 100으로 낮추세요.

변경 요약
config.py 변경 (4개 멤버)
멤버	변경 내용	이유
M1 SPY	algorithm: STATIC→DDPG, ε: 0.08→0.15 (OU noise)	이진 BUY/CASH 한계 → 연속 포지션 [0,1]으로 부분 포지션 가능
M3 KOSPI	use_vol: False→True (8-state)	BUY율 95% → 고변동성 구간 CASH 전환 기회 확보, MDD 개선 목표
M5 SCHD	algorithm: STATIC→A2C	EMA/Bull 4-state 신호 무의미 → 신경망 5D 특징으로 대체
M6 RGLD	algorithm: STATIC→DDPG, ε: 0.1509→0.20 (OU noise)	5시드 동일값(seed 무효) → OU noise 탐색으로 다양성 복원, 원자재 사이클에 부분 포지션 적합
app.py 변경 (레이블 동적화)
DDPG/A2C 사용 시 "STATIC RL"로 표시되던 6곳 수정:

지표 카드: "STATIC RL" → eff_algorithm (예: "DDPG", "A2C")
Action Frequency 차트 제목: "STATIC: Action Frequency" → "DDPG: Action Frequency"
Action 테이블 컬럼 헤더: "STATIC Action/Return(%)" → 알고리즘명 동적 반영
주의 사항
epsilon 파라미터 의미가 바뀜: DDPG에서는 ε-greedy가 아닌 OU noise 스케일
M5 SCHD의 use_vol=True는 A2C에서 무효(신경망은 tabular state 무관) — Vanilla RL 쪽에만 영향
알고리즘 변경 후 Simulation 재실행으로 파라미터 재최적화 권장

Q. 로컬로 시뮬레이션하여 저장된 파라미터들이 다시 웹상이서 그대로 반영되나요?

아니요, 자동으로 반영되지 않습니다.

이유
파라미터 저장 방식을 보면:

_save_sim_params_to_config() (app.py:175)가 members/member_N/config.py 파일을 직접 덮어씁니다
즉 저장 = 로컬 파일 수정
웹(클라우드)은 배포된 코드 기준으로 실행되므로, 로컬에서 파라미터를 저장해도 웹에는 반영되지 않습니다.

웹에 반영하려면
로컬 시뮬레이션 후 파라미터가 마음에 들면:


git add members/member_1/config.py  # 변경된 config.py만 추가
git add members/member_2/config.py  # ...
git commit -m "update optimal params"
git push
→ 웹 앱이 재배포되면 새 config.py 기준으로 실행됩니다.

정리

로컬 시뮬 → config.py 저장 → git push → 웹 반영
로컬과 웹이 같은 config.py를 공유하려면 커밋 푸시가 필수입니다.
