01 · 기초 프레임워크

Markov Decision Process (MDP)

강화학습의 모든 수식은 이 MDP 5-튜플 프레임워크 위에 세워진다. 에이전트는 환경을 관측하고, 행동을 선택하며, 보상을 받고, 새로운 상태로 전이한다.

MDP 정의 — 5-튜플
MDP = ( S,  A,  P,  R,  γ )
S · State (상태)
에이전트의 현재 위치
트레이딩: EMA 위치 × 추세 방향 조합으로 시장 국면을 표현
s ∈ {0,1,2,3} 4개 이산 상태
use_vol=True → s ∈ {0..7} 8개 상태
A · Action (행동)
에이전트가 선택할 수 있는 행동
매 시점 이진(Binary) 결정. BUY=1이면 전액 매수, CASH=0이면 현금 보유
A = {0: CASH(현금보유), 1: BUY(매수)}
P · Transition (전이확률)
P(s'|s,a) — 다음 상태 도달 확률
상태 s에서 행동 a를 선택했을 때 다음 상태 s'로 전이할 확률. 트레이딩에서 시장 가격 변동이 이를 결정하므로 직접 알 수 없다(Model-free)
시장 가격 변동이 외생적으로 결정
R · Reward (보상)
R(s,a,s') — 즉각적 피드백 신호
행동의 즉각적 결과. 수수료를 차감한 순 수익률로 계산
daily_return × a − fee × |a − prev_a|
γ · Discount (할인인수)
미래 보상의 가중치 [0, 1]
γ → 1: 장기적 수익 중시 (장기 투자자)
γ → 0: 눈앞의 보상에 집중 (단기 투기)
실전: γ = 0.87 ~ 0.97 (종목별 최적화)
4가지 이산 상태 설계
상태 공간 설계 공식
state = is_bull × 1 + is_above_ema × 2  [+ is_high_vol × 4]

is_bull = (daily_return > 0) ← 당일 수익률이 양수이면 True
is_above_ema = (Close >= EMA_10) ← 현재가가 10일 EMA 이상이면 True
is_high_vol = (rolling_std > median) ← 변동성이 훈련구간 중위수 초과 시 True
State 추세 EMA 위치 시장 신호 θ[s,1] 초기값 (Cold-Start)
0 하락 (ret ≤ 0) EMA 아래 가장 보수적 0 (매수 신호 없음)
1 상승 (ret > 0) EMA 아래 주의 단계 max(0.05, fee × 30)
2 하락 (ret ≤ 0) EMA 위 중립 max(0.10, fee × 50)
3 상승 (ret > 0) EMA 위 가장 강한 매수 max(0.20, fee × 80)
Cold-Start 수수료 장벽 완화
수수료가 높을수록 BUY 선호 초기화값을 크게 설정하여 초반 CASH 고착을 방지한다. 미국(fee=0.001): θ[1,1]=0.05 / 한국(fee=0.0023): θ[1,1]=0.069. 8-State 확장 시 상태 {4..7}은 고변동성 레짐으로, 동일 EMA·추세 신호에서 별도 정책을 학습한다.
02 · 가치 함수 정의

Bellman Equation

핵심 질문: "현재 상태 s에 있을 때, 앞으로 받을 총 보상은 얼마인가?" — 이 질문의 답이 가치 함수 V(s)이며, 모든 RL 알고리즘의 수학적 토대가 된다.

상태 가치 함수 V(s) — Bellman 기대 방정식
V(s) = E[ r(t) + γ · V(s(t+1)) ]
V(s)
현재 상태의 가치 (State Value)
상태 s에서 시작하여 정책 π를 따랐을 때 받을 총 기대 수익. 에이전트가 학습하고 저장하는 핵심 값. Critic이 담당.
r(t)
즉각적 보상 (Immediate Reward)
행동을 취한 직후 받는 보상. 시장 데이터가 수식에 유입되는 유일한 통로.
= daily_return × a − fee × |a − prev_a|
γ · V(s(t+1))
할인된 미래 가치
다음 상태의 가치를 γ로 할인. 현재의 가치는 미래 가치의 재귀적 합계임을 표현.
γ=0.95: 미래 1봉 후 가치의 95%만 인정
재귀적 구조 (Recursive Structure)
V(s)는 V(s')에 의존한다. 즉 현재의 가치는 미래의 가치로 정의된다. 이 재귀 관계가 동적 프로그래밍과 TD Learning의 핵심 원리이며, 가치 함수를 반복적으로 갱신하면 참값에 수렴한다.
Bellman 최적 방정식 — Q*(s, a)

"상태 s에서 행동 a를 취할 때, 가능한 최선의 가치는 얼마인가?"

최적 행동-가치 함수 Q*(s,a)
Q*(s, a) = E[ r + γ · maxa' Q*(s', a') ]
Q*(s, a)
최적 행동-가치 함수
V(s)와 달리 (상태, 행동) 쌍의 가치를 모두 지정. "상태 s에서 행동 a를 하고, 이후 최적으로 행동하면 총 보상은?"
maxa' Q*(s',a')
미래 최적 행동 선택 (greedy)
다음 상태에서 가능한 행동 중 가장 좋은 것을 선택. 이것이 "최적"의 의미: 미래 결정은 항상 완벽하다고 가정.
V(s) vs Q(s,a)
관계식
V(s) = 상태의 가치 (모든 행동 평균 혹은 최적 행동)
Q(s,a) = 특정 (상태, 행동) 쌍의 가치
V(s) = max_a Q(s,a) (최적 정책 하에서)
03 · Vanilla RL 구현

Q-Learning — Temporal Difference Update

전이 모델 P(s'|s,a)를 모르는 상태에서 Q*를 근사하는 Model-free 방법. 경험(r, s, a, s')을 통해 Q-테이블을 반복 갱신한다.

Q-Learning TD 갱신 규칙 — base_agent.py _train_qlearning_vanilla()
Q(s,a) += α · [ r + γ · maxa' Q(s',a') − Q(s,a) ]
r + γ · maxa' Q(s',a')
TD Target (목표값)
에이전트가 "받았어야 할" 기대값. 실제 보상 r과 다음 상태 최선 행동 가치 추정을 합산. Q(s,a)가 수렴하려는 '정답'.
Q(s, a)
현재 추정값
에이전트가 "예상했던" 값. Q-테이블에 저장된 현재 (상태, 행동) 쌍의 가치. 2×2 행렬: Q[상태][행동]
δ = Target − Q(s,a)
TD Error (예측 오차 신호)
δ > 0: 현실이 예상보다 좋았음 → Q 증가
δ < 0: 현실이 예상보다 나빴음 → Q 감소
α (Learning Rate)
학습률 [0.005 ~ 0.1]
경험 하나당 Q를 얼마나 조정할지.
작은 α = 신중한 학습 / 큰 α = 공격적 학습
실전: α = 0.010 ~ 0.073 (시뮬레이션 탐색)
프로젝트 구현 상세 — Cold-Start 특수 처리
Q-table 초기화 (fee 비례 BUY 선호)
q_table = np.zeros((2, 2))
q_table[:, 1] = max(fee_rate * 50, 0.05)
# 미국 fee=0.001 → Q[:,1] = 0.05
# 한국 fee=0.0023 → Q[:,1] = 0.115
Epsilon Annealing (탐험률 감소)
eps = ε × max(1.0, 2.0 − ep/(T−1))
# 학습 초기: 2×ε → 학습 후기: ε
prev_action = 1 # BUY로 시작 (편향 제거)
훈련 후 보정 — Q-floor margin
Q[0,1] = max(Q[0,1], Q[0,0] + 0.005)
Q[1,1] = max(Q[1,1], Q[1,0] + 0.005)
# 훈련 노이즈로 인한 CASH 고착 방지
# 전 상태에서 BUY 상대 우위 보장
Vanilla의 한계
Q-테이블은 기대값만 저장, 확률 분포를 다루지 못함 → Policy Gradient 불가, 엔트로피 제어 불가. 2-상태만 사용하여 EMA 위치 신호를 버림.
03.5 · 논문 기반 개념 정의

STATIC이란 무엇인가?

Su et al. (2026) "Vectorizing the Trie" 논문에서 제안된 STATIC은 Sparse Transition Matrix-Accelerated Trie Index for Constrained Decoding의 약자로, LLM 기반 생성 검색에서 TPU/GPU 위에 효율적인 제약 디코딩을 실현하는 기법이다. 이 개념은 트레이딩 강화학습의 상태 제약 구조와 구조적으로 동형(isomorphic)이다.

논문이 해결하는 문제
표준 오토레그레시브 디코딩은 출력 공간을 제어할 수 없어 "할루시네이션" — 재고에 없거나 너무 오래된 아이템의 ID를 자신 있게 생성 — 이 발생한다. Trie(프리픽스 트리) 기반 제약은 개념적으로 올바르지만, 포인터 체이싱 특성상 TPU/GPU에서 심각한 지연을 유발한다.
논문 STATIC 핵심 아이디어 — Trie → CSR 행렬 변환
기존 Trie (문제)
포인터 체이싱으로 비연속 메모리 접근 → HBM burst 활용 불가

동적 제어 흐름으로 XLA 정적 그래프 컴파일 불가

CSR 전이 행렬 정의 (논문 수식 4.2)
T[s, v] = s_next  if  (s →v s_next) exists,   else 0

s : 현재 prefix 노드 상태 (trie node index)
v : 선택한 토큰 (token index)
s_next : 전이 후 다음 노드 ID
0 : 무효 전이 (터미널 상태 → 해당 토큰 마스킹)
CSR 3개 배열 구조 + VNTK 슬라이싱
P[n] : Row Pointers       ← 노드 n의 유효 전이 시작 인덱스
C[P[n]:P[n+1]] : Columns ← 유효 토큰 ID 목록
V[P[n]:P[n+1]] : Values ← 대응하는 다음 노드 ID 목록

# VNTK (Vectorized Node Transition Kernel) 핵심 연산
d_col = DynamicSlice(C, start=P[n], len=B_t) ← 고정 크기 슬라이싱
m = Scatter(valid_tokens, values=True) ← 희소 → 밀집 마스크
log_prob[invalid] = −∞ ← 무효 토큰 제거
트레이딩 STATIC RL과의 구조적 대응

논문의 STATIC과 트레이딩 STATIC_H는 서로 다른 도메인의 동일한 수학 구조를 공유한다. 논문은 LLM 출력 공간을 유효 아이템으로 제약하고, 트레이딩 시스템은 시장 상태를 유효 행동으로 제약한다.

개념 논문 STATIC (LLM 디코딩) 트레이딩 STATIC_H (RL)
노드/상태 Trie prefix 노드 s ∈ [S] 시장 상태 s ∈ {0,1,2,3} or {0..7}
선택/행동 SID 토큰 v ∈ V 행동 a ∈ {0:CASH, 1:BUY}
전이 구조 T[s,v] = s_next (CSR 행렬) θ[s,a] = 선호도 (정책 행렬)
무효 처리 log_prob[invalid] = −∞ π(a|s) → 0 (낮은 θ로 억제)
제약 조건 freshness / category / 재고 EMA 위치 / 추세 방향 / 변동성
효율화 기법 CSR O(1) 조회 + PPO Clip Tabular O(1) 조회 + PPO Clip
안정화 PPV → STATIC (947× 개선) STATIC → STATIC_H (전 종목 양수 Alpha)
시장 정보 학습에 STATIC_H가 유리한 이유
① 제약된 상태 공간
4/8개 이산 상태만 학습 → Cold-Start 조건(사전 데이터 없음)에서도 빠른 수렴. 논문의 CSR처럼 유효한 시장 국면만 탐색.
② Adaptive Temperature
flip_rate 기반 α_t로 시장 레짐 변화를 자동 감지. 상승장→하락장 전환 시 탐험을 즉시 늘려 OOS 구간 재적응 속도 향상.
③ Tabular PPO Clipping
논문의 PPO 클리핑 개념을 Tabular 정책에 적용. 단기 노이즈에 의한 정책 급변(BUY 100% 고착)을 방지하여 OOS 안정성 확보.
04 · STATIC RL — Actor-Critic

Policy Gradient + TD Critic 이중 구조

STATIC은 4/8-상태 Tabular Actor-Critic 알고리즘이다. Actor가 확률적 정책으로 행동을 선택하고, Critic이 TD 오차로 상태 가치를 평가한다. Q-Learning이 기대값만 저장하는 것과 달리, STATIC은 행동 확률 분포를 직접 학습한다.

🎭 Actor (정책 π)
역할: 어떤 행동을 취할지 결정
파라미터: θ[s, a] (선호도 행렬, shape=(4,2) or (8,2))

Softmax 정책:
π(a|s) = exp(θ[s,a]) / Σ exp(θ[s,:])
θ[s, BUY] > θ[s, CASH] 이면 BUY 확률이 더 높음. 원시 선호도를 행동 확률 분포로 변환.
🧠 Critic (가치 V)
역할: 현재 상태가 얼마나 좋은지 평가
파라미터: V[s] (가치 테이블, shape=(4,) or (8,))

TD(0) 갱신:
δ = r_eff + γ·V(s') − V(s)
V(s) += lr · δ
δ = TD Error = 예측 오차 신호. 양수: 실제>예상 / 음수: 실제<예상. V(s)는 누적 시장 지식 저장소.
공유 신호 δ (TD Error)
Critic의 TD Error δ = r_eff + γ·V(s') − V(s) 가 Actor와 Critic 양쪽에 전달된다. Actor에게는 "이 결정이 예상보다 좋았는지 나빴는지"를, Critic에게는 "가치 추정이 얼마나 틀렸는지"를 알려준다.

05 · 탐험 제어

Entropy Regularization — 정책 붕괴 방지

탐험 제어 없이는 정책이 항상 BUY 또는 항상 CASH로 붕괴(고착)될 수 있다. 엔트로피를 보상 신호에 더해 에이전트가 다양한 행동을 유지하도록 유도한다.

엔트로피 정의 H(π)
H(π) = − Σa π(a|s) · log π(a|s)
Entropy H(π) 직관적 이해
정책의 행동 선택 불확실성(무질서도) 측정치. 높은 엔트로피 = 다양한 행동 선택 = 더 많은 탐험.
π = (0.50, 0.50) — 최대 불확실성
H = 0.693
π = (0.75, 0.25) — 중간 확실성
H = 0.562
π = (0.99, 0.01) — 거의 확실
H = 0.056
π = (1.00, 0.00) — 완전 결정적 (붕괴)
H = 0.000
엔트로피 정규화 보상 — STATIC에서 고정 계수 0.05 사용
r_eff = reward + 0.05 · H(π)

reward = daily_return × a − fee × |a − prev_a|
H(π) = −Σ π(a|s) · log π(a|s) ← 현재 정책 엔트로피
엔트로피 보상의 효과
계수 0.05는 STATIC에서 고정값이다. 시장이 일관되게 BUY를 보상하더라도, 엔트로피 보너스가 정책 붕괴(π(BUY)=1.0)를 방지한다. STATIC_H에서는 이 계수가 시장 상황에 따라 자동으로 변한다(Adaptive Temperature).

STEP 01 — 수학적 기초
Bellman Equation
V(s) = E[r + γ·V(s')]

상태 가치의 재귀적 정의. 현재 가치 = 즉각 보상 + 할인된 미래. 모든 RL의 수학적 토대.

Q-Learning (Vanilla RL)
Q(s,a) += α·[r+γ·maxQ(s',a')−Q(s,a)]
Tabular Q값, 2-상태. Bellman 최적 방정식 기반 Off-policy TD 갱신. Cold-Start + Q-floor Margin 보정.
STEP 03+04 — 확률적 정책
Actor-Critic (STATIC)
θ += lr·δ·(1[a]−π)
r_eff = R + 0.05·H(π)
Policy Gradient + TD Critic, 4/8-상태. Softmax 확률 정책 + 엔트로피 보너스 (고정 0.05). 논문의 CSR 노드처럼 시장 상태 제약.
STEP 05 — 하이브리드
STATIC_H (Hybrid)
α_t = clip(0.05·(1+flip_rate), 0.01, 0.15)
θ += scale·lr·δ·grad_log_π

10 · 기승전결 종합 정리

Bellman에서 Team Fund +42.58%까지

기 (起) — 문제 정의

왜 강화학습이 필요한가?

전통적 트레이딩 전략은 고정 규칙으로 작동한다. 그러나 시장은 상승장·하락장·고변동성·저변동성이 끊임없이 교체되는 비정상(non-stationary) 환경이다. 사전 데이터 없는 Cold-Start 조건에서 에이전트가 시장을 스스로 학습해야 한다는 것이 이 프로젝트의 출발점이다. Bellman 방정식 V(s) = E[r + γ·V(s')]은 "현재의 가치는 미래의 가치로 정의된다"는 재귀적 진리를 수학적으로 표현하며, 모든 RL의 토대가 된다.

승 (承) — 구조 설계

어떻게 시장을 상태로 인코딩하는가?

시장을 state = is_bull×1 + is_above_ema×2로 4개(또는 8개) 이산 상태로 압축한다. 이는 Su et al.(2026) 논문의 CSR 전이 행렬 구조와 동형이다 — Trie 노드가 시장 상태로, SID 토큰이 BUY/CASH 행동으로 대응된다. Q-Learning으로 TD 오차를 학습하고, STATIC Actor-Critic으로 확률적 정책 π(a|s) = softmax(θ[s])를 직접 학습하며, 엔트로피 보너스 r_eff = R + 0.05·H(π)로 정책 붕괴를 방지한다.

전 (轉) — 핵심 혁신

STATIC_H가 STATIC을 어떻게 넘어서는가?

STATIC의 두 한계 — 고정 엔트로피 계수(0.05)와 갱신 크기 제약 없음 — 를 극복하기 위해 두 메커니즘을 추가한다. Adaptive Temperature: α_t = clip(0.05×(1+flip_rate), 0.01, 0.15)로 시장 전환 빈도에 따라 탐험 강도를 자동 조절한다. Tabular PPO Clipping: |r_t−1|>0.2 → scale=0.3으로 정책 급변을 억제하여 OOS 안정성을 확보한다. 이는 논문(Su et al. 2026)의 948× 속도 개선과 같은 구조적 혁신이 다른 도메인에서 반복되는 사례다.

결 (結) — 실증 성과

결과는 수식이 증명한다

워크포워드 검증(앞 70% 학습 / 뒤 30% OOS)으로 과적합을 방지한 상태에서, STATIC_H는 6개 종목 전체에서 시장 대비 양수 Alpha Gap을 달성했다. Vanilla Q-Learning 대비 풍부한 4/8 상태 공간, 확률적 정책, 동적 엔트로피, PPO 클리핑의 복합 효과가 Team Fund +42.58%라는 단일 숫자로 수렴한다. Bellman의 재귀 공식 하나에서 출발하여 실전 트레이딩 시스템까지 — 모든 것은 연결된 하나의 수학적 체계다.
